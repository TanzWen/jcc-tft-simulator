# AGENT.md

JCC 项目通用智能体协作规范。所有参与本仓库工作的自动化代理均应先阅读本文件；Claude 还应同时遵守 `CLAUDE.md`。

## 工作边界

1. 修改代码、配置、文档、数据库或系统状态前，先用简短文字说明方案、影响文件和验证方式，等待用户确认。
2. 阅读、搜索、查看 Git 状态/历史/diff 等只读操作可以直接执行。
3. 用户明确授权“直接改”“不用问”或“一步到位”时，当次任务可直接实施。
4. 发现工作范围需要明显扩大时，暂停修改并重新确认。
5. 不擅自提交、推送、建分支、创建 PR，也不清理用户已有改动。

## Git 规则

- commit message 禁止加入 `Co-Authored-By: Claude ...` 或任何 Claude 共作署名。
- commit 前检查 `git status` 和待提交 diff，避免混入无关文件。
- commit 前同步检查并更新 `CLAUDE.md` 与 `README.md`。
- commit 前运行与改动相匹配的测试；测试失败不得以完成状态提交。

## 新项目规则

若任务是在空目录中创建新项目，必须同时：

- 创建适合技术栈的 `.gitignore`。
- 创建 `CLAUDE.md` 和 `README.md`。
- 创建 `docs/` 用于方案文档。
- 将 `docs/` 加入 `.gitignore`，默认不提交方案文档。

## 项目概览

- 目标：采集金铲铲之战官网数据，进行赛季版本管理、数据分析、理论强度评分和阵容生成。
- 语言：Python 3.11+，使用 Conda base 环境的 `python`。
- 依赖：运行时仅使用标准库。
- 存储：SQLite，默认 `data/jcc.db`。
- 前端：原生 HTML/CSS/JavaScript，液态玻璃 UI。
- 数据源：金铲铲之战官网及腾讯官方 CDN 公开资源。

## 核心模块

```text
src/jcc/collector.py   官网配置、版本索引和结构化数据采集
src/jcc/database.py    SQLite schema、迁移和事务入库
src/jcc/images.py      头像与 1624×750 横版原画下载
src/jcc/cli.py         数据更新入口
src/jcc/web/server.py  本地 API、静态页面和图片服务
src/jcc/web/static/    前端页面
tests/                 unittest 测试
```

`data/raw/`、`data/assets/` 和 `data/*.db` 均为可再生本地产物，不应提交。

## 实施规则

### 采集器

- 动态解析官网当前 mode、season、version 和图片模板，不硬编码当前版本。
- 保留原始快照、来源 URL 和 SHA-256。
- 对网络断流、无效 JSON、无效图片进行重试或安全失败。
- 不完整采集不得覆盖已有完整数据库。

### 赛季更新

- 更新到新赛季或新版本只需显式运行 `PYTHONPATH=src python -m jcc.cli`，不改代码：采集器读官网 `basicConfig.js` 取当前 mode 与赛季，再从版本索引中选出当前日期已生效的最新版本。
- `start.sh` 不承担版本追新职责，它只在数据库缺失或缺字段时兜底采集；不得把它当作更新入口，也不得为了追新而扩大它的判断逻辑。
- 常用变体：`--skip-images` 只更新结构化数据；`--version` 抢先采集尚未生效的版本；`--season` 搭配 `--version` 回溯旧赛季；`--mode` 切换玩法。
- 更新后数据按 `season_id` 隔离，旧赛季保留；原始快照与图片按 `mode<模式>_<赛季>_<版本>` 分目录，互不覆盖。
- 采集失败不写入半份数据。若报错或数量明显异常，先用 `data/raw/` 下的快照定位官网字段变化，再调整 `collector.py` 的 `transform_*` 函数。
- 新赛季若出现新的羁绊配色（`color` 超出 1–5）或新的装备类型，前端筛选与徽章按数据动态生成，但 `styles.css` 的配色变量只定义到 `--tier5`，需要补充对应色值。

### 数据库

- 赛季数据必须隔离。
- schema 变更必须包含已有数据库迁移。
- 同步更新所有写入、查询、API、测试、README 和启动检查。
- 完成后检查 SQLite 外键与完整性。

### 图片

- 区分头像与横版原画，分别保存官网 URL 和本地路径。
- 横版原画使用官网 `hero_pic_big` 模板生成，不能由头像放大或裁切生成。
- 下载完成后校验格式，并确保所有数据库路径均指向真实文件。

### 前端

- 数据来自本地 API，避免重复维护静态数据。
- 英雄池优先显示本地横版原画；小尺寸棋盘位可使用本地头像。
- 羁绊统计、过滤和阵容状态应保持确定性、可复现。
- 理论评分必须展示评分依据，并明确不等同于真实对局胜率。
- 列表与棋盘重绘必须复用已有节点：`renderAll` 只同步状态，`replaceChildren` 会把复用节点 detach 再 attach 导致图片重绘，统一走 `syncChildren` 先比对再替换。
- 装备只汇总 `basic_description` 中的基础属性；转职纹章按 `equipment.fetter_id` → `traits.trait_key` 追加羁绊并计入统计。

## 常用命令

```bash
./start.sh                                             # 一键启动，必要时兜底采集
PYTHONPATH=src python -m jcc.cli                       # 更新到官网当前已生效版本
PYTHONPATH=src python -m jcc.cli --skip-images         # 只更新结构化数据
PYTHONPATH=src python -m jcc.web.server --open         # 只启动本地页面
PYTHONPATH=src python -m unittest discover -s tests -v # 运行测试
```
